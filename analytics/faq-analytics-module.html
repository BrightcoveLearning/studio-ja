1.アナリティクスモジュールとは何ですか？

Video Cloud Studio Analytics モジュールは、動画やプレーヤーに関するリアルタイム分析情報をほぼリアルタイムで表示するために使用できるユーザーインターフェイスモジュールです。Analytics プラットフォームは、ビッグデータ技術の最新イノベーションを使用し、大規模に設計されています。分析プラットフォームは、分析データへの迅速かつほぼリアルタイムアクセスを提供し、開発者は Video Cloud Studio に含まれるものに加えて、カスタム分析ソリューションやレポートを作成できる強力なオープン API を備えています。

2.分析モジュールからレポートデータをエクスポートできますか？

はい分析レポートには、.csv ファイルおよび.xlsx ファイルにレポートデータをダウンロードするために使用できるリンクがあります。

3.分析モジュールでデータが更新される頻度はどれくらいですか？

通常、リアルタイムに近いデータは約 15 分遅れます。ただし、この時間は、データ収集システムが現在受信しているデータの量によって異なる場合があり、場合によっては最大3時間になる場合があります。

Analytics システムは複数のソースからデータをコンパイルし、リアルタイムに近いデータは、これまでに収集されたデータに基づく近似値です。数値は、レポートごとに異なる場合があり、ハードな数字ではなく傾向として見るべきです。

4.レポートやAPIに内部動画 ID を含める方法はありますか？

ビデオプロパティとして使用できる [参照 ID] フィールドを使用すると、内部 ID を Video Cloud の動画にマップできます。[参照 ID] フィールドは、カスタムレポートの作成時に表示できます。カスタムレポートの詳細については、「Analytics モジュールでのカスタム Analytics レポートの作成」を参照してください。

5.動画の前にプレロール広告があり、視聴者が数秒間広告を見てから離れた場合はどうなりますか。視聴者がメインコンテンツをまったく視聴しなかった場合、それは動画再生数としてカウントされますか？

なし動画視聴回数は、メインコンテンツの再生が開始された後にのみカウントされます。

6.分析から IP アドレスの範囲をフィルタリングすることは可能ですか？

現時点では、IP アドレスの範囲をフィルタリングする機能は使用できません。

7.アナリティクスはライブイベントストリーミングでプレイヤーを追跡しますか？

Analytics トラックは、パフォーマンスレポートの一部として表示時間（分）。カスタムレポートを作成するとき、または請求モジュールを使用して、ライブ表示秒数を表示できます。ライブイベントの帯域幅は表示されません。

8.分析モジュールを使用して特定の動画に関する情報を見つけるにはどうすればよいですか？

検索機能は、特定の動画を検索するために使用することができます。動画名、短い説明、詳細説明、タグ、動画 ID、参照 ID、およびカスタムテキストフィールドが検索されます。結果をクリックすると、その動画のパフォーマンスまたはエンゲージメントが表示されます。カスタムレポートを作成して、特定の動画 ID の分析を表示することもできます。

9.分析モジュールの参照 ID を使用して特定の動画を検索できますか？

はい検索機能を使用すると、参照 ID フィールドに基づいて動画を検索できます。動画名、短い説明、詳細説明、タグ、動画 ID、およびカスタムテキストフィールドも検索されます。

10.顧客エンゲージメントと特定の動画からのドロップオフについてどのようにレポートできますか？

エンゲージメントレポートには、この情報が表示されます。エンゲージメントレポートを選択し、動画ディメンションを選択すると、アカウントのエンゲージメント全体を示すグラフが表示されます。動画を選択すると、その動画のエンゲージメントが時間の経過とともに表示されるグラフが表示されます。

11.URLで報告することはできますか？たとえば、ページ A とページ B で同じ動画をホストしている場合、2 つのプレースメントのパフォーマンスを比較できますか。

はいパフォーマンスレポートでは、ディスプレイドメイン別に動画分析を表示できます。

12.ビデオがループしている場合、ビデオが再生されるたびに 1 つのビデオビューまたは複数のビューとしてカウントされますか？

ビデオビューは、ストリームの開始時に、視聴セッションごとに 1 回だけカウントされます。エンゲージメントレポートの x% での視聴率は、パフォーマンスレポートのビデオビューと同じではないことに注意してください。動画再生回数は、動画の再生開始時にカウントされます。1% の視聴率は、最初のパーセントが視聴されたときに記録されます。ここでの違いは、最初の割合が複数回視聴された場合（例えば、ビデオが自動再生に設定されている場合）、カウントが増加します。一方、ビデオビューは、ビデオが最初に再生を開始したときにカウントされますが、再生にはカウントされません（ユーザーがページをリロードするか、プレイリストなど）。

13.レポートを実行して、自分のアカウントの一意の閲覧者数を表示できますか？

個別閲覧者の数は、パフォーマンスレポートに表示されます。

14.Analytics モジュールで使用可能な標準レポートをカスタマイズできますか。

パフォーマンスレポートとエンゲージメントレポートに表示されるデータはカスタマイズできません。ただし、Analyticsモジュールでは、カスタム解析レポートを作成することができ、表示するデータを選択できます。もありますAnalytics APIプログラムで分析データにアクセスするために使用できます。

15.エンゲージメントスコアが非常に高く、100を超えているのはなぜですか？

エンゲージメントスコアは 100 より大きくなる可能性があります。ビデオをスクラブして再視聴しても、新しい video_view イベントはトリガーされませんが、video_engagement がトラッキングされます。たとえば、ページ上の単一のビデオプレーヤーにビデオがあるとします。以下のイベントは、動画エンゲージメントが大きく異なることを示しています。

  1. 時計はまだありません。エンゲージメントスコア:0.
  2. ビューアはページをロードし、ビデオをすべて見ます。エンゲージメントスコア:100 (100% 表示/1 ビュー)
  3. 同じビューアがスクラブしてもう一度見ます（ブラウザはリロードされませんでした）。エンゲージメントスコア:200 (200% 閲覧/1 ビュー)。
  4. 別のビューアがページをロードし、ビデオ全体を見ます。エンゲージメントスコア:150 (300% 閲覧回/ 2 ビュー)

ビデオエンゲージメントの計算方法の詳細については、トレーニングビデオ「ビデオエンゲージメントの計算」をご覧ください。

16.ビューの指標が 1% のビュー数より 25% 高いのはなぜですか？

2 つの数値に不一致を引き起こす可能性のある要因がいくつかあります。

  • 1% のビューと x% でのビューは 2 つの異なる指標です。1% のビューは、表示セッションごとに一意であり、巻き戻しと再生は含まれません。視聴者がビデオの前の部分に再生または巻き戻すと、% ポイントは視聴者数を増やしますが、動画の再生数は増加しません。
  • % のビューとエンゲージメント曲線は、UTC 時間を使用して毎日の粒度で保存されるため、UTC とレポートのタイムゾーンがクロスオーバーする場合、特定の日付に 2 日分のデータが含まれることがあります。

17.特定の日の動画再生回数が時間の経過とともに変化するのはなぜですか？

分析システムは、1 時間ごとに収集された未調整の最近の指標を提供し、日次レベルで報告された実際の指標と自動的に調整されます。この和解は、通常3〜5日で、できるだけ早く起こります。時間単位で収集されたデータを調整することはありません。パブリッシャーは時間ごとのデータを評価しているため、時間ごとの不一致のデータを 32 日間利用できます。和解に関するいくつかの詳細:

  • プレイヤーはデータ収集 API にデータを送信する
  •  データ収集 API は、 リアルタイムデータベースへの書き込みに最適であり、イベントをログストレージに書き込む
  • CDN やその他のソースから追加データを収集すると、分析データが更新されます。これは調整プロセスです。
  • 夜間のバッチシステムは、生のログイベントを調べて、履歴データインポートファイルを構築する
  • データインポートファイルは、履歴データベースにロードされます。API 呼び出しを行うときに、調整済みデータのみを表示するか (最も正確な)、リコンサイルパラメータを使用して調整データと未調整データの両方を表示するかを選択できます。詳細については、 Analytics APIの概要を参照してください。
  • すべての履歴データは調整されます。リアルタイムデータ (32 日間使用可能) は、まだリコンサイルされている場合とされていない場合があります。データを取得している期間が 32 日以上前に終了する場合は、reconciled data, since the data will all be historical.

18.一部のメトリックスがNULL値を返すか、値がないのはなぜですか？

これは、メトリックスがリクエストされた期間に使用できない場合に発生します。メトリックスによっては 2 ～ 3 日間のレイテンシーがあります。そのため、たとえば、過去 30 日間（今日を含む）にこのデータを要求した場合、要求範囲にはまだこのデータがない日が含まれているため、これらのメトリックスは NULL になることがあります。データを要求するときにこれを回避するには、要求された範囲がレイテンシー期間外であることを確認してください。

また、時間の経過とともに指標とディメンションの組み合わせも追加されました。たとえば、複数年のレポートを実行している場合、レポート期間全体でその指標を追跡しなかったため、一部の指標が空白または一部になることがあります。有効な日付とディメンションの組み合わせについては、 Analytics APIの概要ドキュメントを参照してください。

19.2013 の宛先ドメインデータが他のパフォーマンスデータと一致しないのはなぜですか？

宛先ドメイン機能は 2014 年 1 月まで有効になっていませんでした。2013年3月から2014年1月までのデータは部分的に埋め戻されたが、この期間で入手できるのは一部の目的地データのみである。2014 年 1 月からのデータが完了しました。

20.削除された動画の分析は含まれていますか？

アナリティクスレポートには、アカウントから削除された動画の分析が含まれます。ただし、Analytics モジュールで削除された動画を検索して、削除された特定の動画に関するレポートを表示することはできません。このようなレポートを取得するには、削除された動画の動画 ID を知り、 Analytics APIを使用してレポートを生成する必要があります。レポートデータをエクスポートすると、削除された動画のデータが、動画 ID または名前を表示せずに表示されることがあります。

21.アナリティクスレポートから従業員のビューを除外できますか？

これを行う簡単な方法はありません。トラフィックが多い動画では、従業員の視聴回数が非常に少ないため、分析への影響は無視できる程度です。しかし、トラフィックの少ない動画では、これを行うことが重要だと思う場合、最も単純な解決策は、プロダクションプレーヤーを複製し、そのコピーのビデオを従業員にテスト/表示させることです。その後、Analytics モジュールのカスタムレポート機能または Analytics API を使用して、フィルターを使用して、プロダクションプレーヤーでのみレポートを作成できます。

22。エンゲージメントのために常に整数が表示されないのはなぜですか？

差込み数値は、小数点で戻すことができます。その理由は、エンゲージメントが正規化されているためです。つまり、「video_percent_vreed* (video_engagement_25/video_engagement_sum)」の比率であるため、クエリで選択した時間範囲に基づいて、正確に分割されない場合に浮動小数点数が表示されることがあります。

23。ビデオIDのみを表示するレポートがあるのはなぜですか、または一部の動画がまったく表示されないのはなぜですか？

期間中に動画の視聴回数がない場合、レポートには動画に関する広範なデータ（例: 動画 ID のみであり、それ以外には何もありません）が表示されません。データがまったくダウンロードされていない場合（つまり、画像データさえない）、ビデオはレポートに表示されません。

24。分析データを取得できる最も早い日はいつですか？

アナリティクスデータの最も早い日付は 2010 年 12 月 28 日です。一部のデータ項目は後で収集されなかったので、データを見ることができる最も早い日付は、あなたが見ているものに正確に依存します。ビデオビューは 2010 年 12 月 28 日まで利用できます。

25。分析モジュールとAnalytics API同じ日付範囲の同じデータを教えてください。

はい。ただし、比較的最近のデータを比較していて、StudioとAPIから異なる時間にデータを取得する場合は、異なる可能性があります。

  1. データは 3 日未満です。3 日未満のアナリティクスデータは暫定的なデータであり、随時変更されることがあります。
  2. 同じ期間のデータは、32日未満であったとき、そして32日後に再び参照されます。32 日後、データは別の「履歴」リポジトリに移動され、その時点で一部のデータ、特に詳細なエンゲージメントデータが破棄されます。つまり、エンゲージメントスコアなどの計算フィールドは、より細かいデータに対して計算が行われているため、わずかに変化する可能性があります。
