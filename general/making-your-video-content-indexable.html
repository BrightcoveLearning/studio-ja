インターネット検索エンジンは、インターネットで利用可能な情報にアクセスするための素早く簡単な方法をエンドユーザーに提供しています。ますます多くのインターネットコンテンツはマルチメディアであるため、ユーザーがそれを発見できるように、検索エンジンによってコンテンツが正しくインデックス付けされていることを確認する必要があります。この記事では、動画コンテンツを SEO を有効にし、さまざまな検索エンジン (Google、Yahoo、マイクロソフト) でインデックスを作成する方法の概要を説明します。

サイトマップと呼ばれる標準的なドキュメントがあり、検索エンジンインデクサーがサイトを調べるときに検索します。このドキュメントでは、サイトで公開されているコンテンツ、そのコンテンツのメタデータ、およびサイト上のコンテンツの場所を検索エンジンに簡潔に説明します。サイトマップは、標準仕様に従った XML ファイルです。

サイトマップには、作成できる（および必要なもの）という 2 つの異なる種類のサイトマップがあります。

  • サイトマップ -www.google.com や search.yahoo.com などの標準的なテキストベースの検索エンジンでコンテンツをインデックスするサイトマップ
  • 動画サイトマップ -video.google.com などのメディア中心の検索エンジンでコンテンツをインデックスするサイトマップ

これらのサイトマップは両方とも、動画コンテンツに関するメタデータをインデックスし、エンドユーザーにリンクを提供します。違いは ここでは以下の条件です。 インデックス付けされるメタデータは、およびから取得されます。 どうやって コンテンツは検索結果に表示されます。

動画コンテンツの適切なSEO戦略には、作成が含まれます どちらも 標準のサイトマップとビデオサイトマップ。優先度の観点から、最初に標準のサイトマップを作成し、次にビデオサイトマップを作成します。Video Cloud CMS API を使用して、手動でコンパイルするのではなく、動画サイトマップを自動的に生成するスクリプトを作成します。

ブライトコーブのラーニングサービスには、CMS API を使用してビデオサイトマップを生成するためのサンプルアプリがあります。

サイトマップの提出の作成の詳細については、 Google のドキュメントを参照してください。

標準サイトマップ

サイトマップは、ここで定義されているサイトマップ仕様に従います。 www.sitemaps.org。サイトマップファイルの目的は、サイト上の URL の一覧を検索エンジンに提供することです。URL に関連付けられたその他の情報は、ページが最後に変更された日時とページが変更される頻度だけです。このインデックスには、コンテンツに関するメタデータがないことに注意してください。サイトマップはページ中心であるため、次のようなモデルを作成する必要があります。 各 ライブラリ内のビデオには、一意のページまたはURLが関連付けられています。ページにさまざまなクエリパラメータを渡すことによって、その動作およびコンテンツを動的に変更できる単一ページを持つことにより、これは達成できます。たとえば、のようなURLがある場合はhttp://www.example.com/video.html?videoId=123、video.htmlページにvideoIdパラメーター ( videoId=123 ) をクエリし、ブラウザに返されたページの内容を変更し、ID 123 の動画に関する情報を格納します。これは、ページがIDを検索し、VideoCloudを使用するアプリケーションのサーバー側で実行されます。 CMS APIビデオに関するメタデータを取得してページに書き込みます。または、動画が公開されているURLをカスタムフィールドに保存して、 CMS APIそれを取得できます。

サイトマップには、changefreqコンテンツが変更される頻度を検索エンジンに伝えるタグもあります。ビデオを頻繁に更新する場合は、その情報をタグまたはカスタムフィールドにも記録する必要があります。 CMS APIそれを返すことができます。多くの動画はめったに更新されないため、その場合は、タグの値をに設定するかyearly、またはこのオプションのタグを完全に省略することもできます。とにかく、サイトクローラがそれに多くの注意を払うという明確な証拠はありません。

この記事の目的は、サイト上のすべてのビデオコンテンツの再生に使用できるランディングページが 1 つあることを前提としています。ページには異なるクエリパラメータが渡され、再生するビデオとサーフェスメタデータのビデオがページに通知されます。たとえば、プレイリスト全体のコンテンツを表示し、プレーヤー内の特定のビデオをキューに入れるページがあるとします。bcpidクエリパラメータでメタデータを表示するプレイリストと、パラメータを使用してプレーヤーに表示するビデオをページに伝えます。bcvidしたがって、私たちがしたいのは、一意のプレイリストとビデオIDの組み合わせごとにURLを作成することです。

作成されるサイトマップの例を次に示します。


<?xml version="1.0" encoding="UTF-8" ?>
 <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
   <url>
     <loc>http://www.example.com/video?bclid=345&bctid=123</loc>
     <lastmod>2015-01-01</lastmod>
     <changefreq>weekly</changefreq>
   </url>
   <url>
     <loc>http://www.example.com/video?bclid=12&bctid=544</loc>
     <lastmod>2015-03-01</lastmod>
   </url>
 </urlset>

サイトマップファイルを作成するには、を使用するスクリプトを作成できますCMS API Video Cloudアカウントの再生リストをウォークスルーするように要求し、各プレイリストについて、含まれているビデオをウォークスルーします。各プレイリストと動画の組み合わせについて、スクリプトは、サイトマップに項目を作成し、プレイリスト/動画の組み合わせに一意のURLを提供します。サイトのサイトマップファイルを作成したら、sitemap.xml結果のファイルを Web サーバーのルートディレクトリに配置します。たとえば、ドメインがの場合www.example.com、サイトマップファイルはで利用できますhttp://www.example.com/sitemap.xml。さらに、サーバー側のアプリケーションでこのファイルを自動的に生成し、ライブサイトでファイルを更新することをお勧めします。

動画サイトマップ

動画サイトマップは、標準のサイトマップファイルと概念的に似ています。動画サイトマップファイルには、アカウント内の動画ごとにエントリがあります。実際、動画サイトマップはサイトマップスキーマをベースとして使用し、動画メタデータに固有のタグを追加します。

以下の表に、Google が動画サイトマップに必要とするタグの概要を示します。

タグ 必須 説明
<loc> Y ビデオのランディングページ（再生ページ、リファラーページ）を指定します。
<video:video> Y ビデオに関するすべての情報を囲みます。
<video:thumbnail_loc> Y ビデオのサムネイル画像ファイルを指す URL。
<video:title> Y ビデオのタイトル。
<video:description> Y ビデオの説明。
<video:content_loc> 依存 実際のビデオメディアファイルを指す URL。注:<video:player_loc>またはを少なくとも 1 つ指定する必要があります<video:content_loc>。
<video:player_loc> 依存 特定の動画のプレーヤーを指している URL。通常、これは <embed> タグの src 要素内の情報であり、<loc> タグのコンテンツと同じであってはなりません。注:<video:player_loc>またはを少なくとも 1 つ指定する必要があります<video:content_loc>。

サポートされているすべてのタグのリストとタグの詳細な説明については、 Google のドキュメントを参照してください。

以下は、サンプルのビデオサイトマップです。

<urlset xmlns = "http://www.sitemaps.org/schemas/sitemap/0.9" xmlns: video = "http://www.google.com/schemas/sitemap-video/1.1">< URL>< loc> http://www.example.com/videos/video_landing_page.html< / loc><ビデオ: ビデオ><ビデオ: thumbnail_loc> http://www.example.com/thumbs/123.jpg< / video: thumbnail_loc><ビデオ: タイトル>夏のステーキのグリル</ video: title><ビデオ: 説明>毎回完璧にステーキを完成させる</ video: description><ビデオ: content_loc> http://brightcove04.brightcove.com/22/20318290001/201510/2249/20318290001_4582301804001_4582289844001.mp4?pubId=20318290001&videoId=4582289844001< / video: content_loc><ビデオ: player_loc> http://players.brightcove.net/20318290001/default_default/index.html?videoId=4582289844001< / video: player_loc><ビデオ: 期間>600< / video: duration><ビデオ: publication_date> 2014-11-05T19: 20: 30 + 08: 00。< / video: publication_date><ビデオ: タグ>ステーキ</ video: tag><ビデオ: タグ>お肉</ video: tag><ビデオ: タグ>夏</ video: tag><ビデオ: family_friendly>はい</ video: family_friendly>< / video: video>< / url>< / urlset>

動画サイトマップを作成したら、こちらの手順に従って Google に送信できます。

注:好みに応じて、標準サイトマップとビデオサイトマップ用に別々のファイルを作成するか、ビデオコンテンツに関する情報を既存のサイトマップドキュメントに追加することができます。ただし、サイトマップファイルには50,000を超えるURLを含めることはできず、50MBを超えることはできないことに注意してください。サイトマップのガイドラインの詳細については、Google のサイトマップ形式とガイドラインのサポート記事をご覧ください。

Brightcove Gallery

ブライトコーブギャラリーでは、技術的なリソースを必要とせずに、魅力的な動画体験を作成できます。ブライトコーブギャラリーを使用して構築されたサイトでは、動画メタデータが自動的に動画サイトマップにまとめられます。詳細については、「ポータル・エクスペリエンスの検索エンジン最適化設定の構成」を参照してください。

SEO スキーマジェネレータプラグイン

動画コンテンツのSEOを増やすもう1つの良い方法は、 SEOスキーマジェネレータプラグインを使用することです。プラグインは、HTML に埋め込まれた動画コンテンツに関する情報を検索エンジンに提供するビデオスキーマのマイクロフォーマットを自動的に生成します。

マイクロフォーマットの詳細、および何らかの理由でプラグインを使用できない場合のビデオスキーマの作成方法については、次のセクションを参照してください。

マイクロフォーマットを使用したコンテキストの追加

ビデオサイトマップに加えて、マイクロフォーマットを使用して、サイト上のビデオコンテンツの可視性を高めることができます。

HTML タグは、コンテンツの表示方法をブラウザに指示しますが、コンテンツが何を意味するかについての情報は提供しません。サイトは構造化データから生成され、データベースに格納されている可能性がありますが、HTML 形式の場合は構造が失われます。検索エンジンは、サイト上のデータの構造を知ることによって利益を得ることができます。

Webページ上のコンテンツの種類を識別するために、スキーマのコレクション（HTMLタグ）が作成されました。Bing、Google、Yahooを含む検索エンジンは、検索結果を改善するために、このマークアップに依存しています。

これらのスキーマ（マイクロフォーマット）は、ウェブページのコンテンツを定義するマークアップに追加する共有語彙のコレクションを提供します。これらのタグの詳細については、 schema.org を参照してください。

メディアプレーヤーでは、 Brightcove では VideoObjectスキーマを使用することをお勧めします。

ブライトコーブは、マイクロフォーマットデータを含むプレーヤー公開コードを生成するツールを作成しました。ビデオスキーマビルダーをチェックしてください。

結論

動画コンテンツの完全な SEO 戦略には、標準サイトマップと動画サイトマップの両方を作成する必要があります。これにより、コンテンツのインデックスが検索エンジンの最も広い範囲にわたって作成されます。さらに、あなたのコンテンツは、発見のポイントのために最も審美的に喜ばれる方法で表示されます。この記事では、アカウント内の動画ごとに一意の URL を提供する方法と、エンジンがインデックスを作成するサイトマップファイルを生成するためのサンプルコードについて説明しています。
