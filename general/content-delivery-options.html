はじめに

Video Cloud アカウントを作成した後に最初に決定すべき点の 1 つは、動画ファイルの配信方法です。Video Cloud には、さまざまな利点と制限がある多くの選択肢があります。選択する配信方法は、動画ファイルの保護度、視聴者へのリーチ方法、アップロード方法、動画アセットが別の CDN（コンテンツ配信ネットワークまたはコンテンツ配信ネットワーク）に保存されているかどうかなどによって異なります。

このトピックで説明するコンテンツ配信オプションは、Video Cloud Enterprise パブリッシャーのみが利用できます。動画クラウドエンタープライズおよび Express パブリッシャーは、CDN としてブライトコーブを使用する必要があります。

CDNって何ですか？

パブリッシャーは、CDN を使用して、オンラインで視聴者に広くメディアを配信します。Wikipedia では、CDNを次のように定義しています。

コンテンツ配信ネットワーク（Content Delivery Network）またはコンテンツ配信ネットワーク（CDN）は、エンドユーザーにパフォーマンス、スケーラビリティ、およびコスト効率を向上させる目的で、コンテンツを最も頻繁に配信するために透過的に協力するコンピュータシステムです。

商用のCDNは多数あります。動画クラウドエンタープライズまたは Express パブリッシャーの場合、ブライトコーブは CDN ソリューションです。Video Cloud Enterprise パブリッシャーの場合は、ブライトコーブ CDN を使用するか、または Video Cloud がサポートする CDN 経由でコンテンツを配信するかを選択できます。

CDN なしでコンテンツを配信することはできますか？

トラフィック量が少ないことが予想される場合、または動画へのトラフィックを完全に制御できる場合は、動画を提供するために CDN は必要ありません。たとえば、一部のパブリッシャーは、自分のコンテンツを内部の Wiki またはプライベートネットワークで配布したいと考えています。システムが、外部 CDN を必要とせずにメディアの配信を処理できる可能性があります。ブライトコーブがこのセットアップをどのようにサポートできるかについての詳細は、ブライトコーブのカスタマーサポートまたはアカウントマネージャにお問い合わせください。

メディアコンテンツはどのように配信されますか？

コンテンツ配信メカニズムにはいくつかのタイプがあります。

  • ストリーミング
  • プログレッシブダウンロード

ストリーミング (HLS または DASH とも呼ばれます)

ストリーミングビデオは、視聴者のコンピュータ/デバイスにダウンロードされることなく、ストリーミングサーバー経由で配信されます。視聴者が再生を押すとすぐに、ビデオの再生が開始されます。ユーザーがビデオのいくつかの他の部分に転送またはスキップすることを決定した場合、彼/彼女はすぐにそれを行うことができ、ビデオはその時点以降から再生され続けます。ストリーミングメディアの利点の 1 つは、ビデオの視聴部分のみが配信されるため、視聴者が視聴したビデオにのみ帯域幅が使用される点です。クライアント側には何も保持されません。すべてがサーバー側にあります。

HTML5ビデオプレーヤーの時代では、すべての最新のデバイスとブラウザーはストリーミングをサポートしており、利用可能な場合は（プログレッシブダウンロードではなく）ストリーミングを使用することに注意してください。これは良いことです。視聴者がビデオ全体を見ないと、バッファリングの管理が改善され、帯域幅を節約できるからです。

ストリーミングは、以下のことを行いたい、またはその必要がある場合に便利です。

  • 長いファイル (10 分以上) または高ビットレートのファイル (高精細) を配信
  • Video Cloud マルチビットレートストリーミング機能を活用してください。この機能は帯域幅検出を実行し、利用可能なハードウェアに対して最高の品質を提供します。プログレッシブダウンロードではマルチビットレートストリーミングを使用できますが、帯域幅の検出は再生の開始時にのみ行われ、プレーヤーは'視聴者の帯域幅の変化に適応するな
  • 帯域幅を節約して、大量のストリームを提供する。
RTMPストリーミングへの参照に遭遇する可能性があることに注意してください。これは古いフラッシュストリーミング形式であり、現在はほとんど使用されていません。

iOS デバイスへのストリーミング

Apple iOS デバイスは、ビデオファイルの配信用に Apple HTTP ライブストリーミング (HLS) と HTTP (プログレッシブダウンロード) のみをサポートしています。動画に利用可能な HLS レンディションが 1 つ以上あることを確認する必要があります。App Store のルールでは、長い形式のビデオコンテンツ（5 MB または 10 分以上）に Apple HTTP ライブストリーミングを使用することが要求されます。

ストリーミングキーポイント:

  • ビデオコンテンツは一時的なビデオリンクを介して保護されます
  • ビデオを進めることは可能です
  • Video Cloud の動的配信機能を効果的に活用
  • より少ない帯域幅で、より多くのステーキを提供することができます
  • Apple HTTP ライブストリーミング (HLS) またはリアルタイムメッセージングプロトコル (RTMP) プロトコルが使用中

実際には、上記のように識別された機能の一部を探さない限り、コンテンツがストリーミングされたか、徐々にダウンロードされたかを認識することはほとんどありません。

プログレッシブダウンロード（PD）

プログレッシブダウンロードが必要なのは、非常に古いブラウザまたはデバイスのみです。この主な用途は、ユーザーがダウンロードして保存できるレンディションを提供することです。

プログレッシブダウンロードでコンテンツを配信する場合は、通常のWebページやダウンロード可能な文書と同様に、標準のWebサーバーからHTTP経由でファイルが提供されます。ビデオを再生すると、ビデオファイルは視聴者のコンピュータまたはデバイスにダウンロードされ、一時ディレクトリに保存されます。十分なファイルがコンピュータ/デバイスにダウンロードされると、ビデオの再生が開始されます。視聴者がビデオの別の部分に早送りまたはスキップしたい場合は、ビデオのその部分が既にダウンロードされて保存されている場合にのみ、再生することができます。ストリーミングビデオと比較すると、プログレッシブダウンロードには一貫したメリットが1つしかありません。ビデオを配信するのにストリーミングサーバーは必要ありません。プログレッシブダウンロードビデオは、通常のWebサーバーから提供することができます。

プログレッシブダウンロードは、トラフィック要件の少ない愛好家や Web サイト、視聴者のコンピュータ/デバイスにコンテンツがキャッシュされているかどうか、および短い長さの動画 (10 分未満) のみを配信する必要がある場合は構いません。動画配信に関する高度な機能と制御が必要なパブリッシャー、および/または多数の視聴者（数百人以上の同時視聴者）に動画を表示する必要があるパブリッシャー、動画の使用状況や視聴統計の追跡と報告が必要なパブリッシャー、または最高のインタラクティブ再生体験を提供したい方はビデオをストリーミングする必要があります。ストリーミング配信では、視聴された動画の一部のみが実際に配信されるため、プログレッシブ配信よりも帯域幅が消費されません。

プログレッシブダウンロードのキーポイント:

  • 限られたシーク機能とナビゲーション機能
  • 閲覧者はコンテンツにアクセスして再利用できる
  • Video Cloud の動的配信機能を効果的に活用できない
  • 安全性が低く、コンテンツを容易にコピー可能
  • 視聴しているものだけでなく、ビデオのすべての部分がダウンロードされるため、より多くの帯域幅が使用されます

画像アセットはどのように配信されますか？

動画コンテンツにストリーミングダウンロードとプログレッシブダウンロードのいずれを使用しているかに関係なく、Video Cloud プレーヤーでエンドユーザーに配信された画像（サムネイルやビデオ静止画など）は HTTP ダウンロードで配信されます。

コンテンツ配信の比較:ブライトコーブがホストする動画アセットと BYO/リモートアセット

Video Cloud では、CDN プロバイダーと設定に応じて、いくつかの基本的な CDN 設定をサポートしています。選択肢は次のとおりです。

  • ブライトコーブ CDN
  • BYOCDN
  • リモートアセット
  • 多肢選択

ブライトコーブCDN

あなたは使用することを選択できますブライトコーブCDN。Video Cloudは、Akamai、Fastly、およびその他のTier 1 CDNを活用して、ビデオコンテンツのストリーミング配信とプログレッシブダウンロード配信の両方を提供します。一般的に HLS または DASH ストリーミングが使用され、プログレッシブダウンロードは特殊な場合のみ、またはビデオのダウンロード可能なレンディションを提供するために使用されます。

BYO CDN

また、独自のCDN（BYO CDNまたは「独自の帯域幅を持参」）を使用することもできます。この場合、CDN プロバイダーとの契約によっては、プログレッシブダウンロード（PD）またはストリーミングメカニズムを使用して、ビデオクラウドプレーヤーを通じてシームレスに配信されます。BYO CDN を選択した場合、Video Cloud のプルベースの取り込みを使用して Video Cloud アカウントにコンテンツを取り込むことができます。その後、コンテンツは、Video Cloud プレーヤーを通じてエンドユーザーに配信するために CDN プロバイダーに移動されます。

リモートアセット

最終的な選択は リモートアセット。 これは、すでに動画ファイルがCDNに格納されており、新しいコンテンツを追加するためにVideo Cloudのアップロードツールを使用する必要がまったくない場合に選択します。Video Cloud Mediaモジュール、Dynamic Ingest API、またはCMS API CDNプロバイダーによってリモートに保存された基盤となるビデオアセットを指すビデオをVideoCloudアカウントに作成します。

制約事項

リモートアセットを使用してビデオを作成する場合、画像キャプチャ機能を使用してリモートアセットビデオの静止画像とサムネイル画像を作成することはできません。動画静止画とサムネイル画像は、リモートアセット動画用にアップロードできます。

リモートアセットの使用方法については、「リモートアセットビデオの操作」を参照してください。

多肢選択

BYO CDN とリモートアセットの両方を使用するように Video Cloud アカウントを設定できます。たとえば、動画コンテンツの一部が CDN で既に使用可能な場合、既存のコンテンツにはリモートアセットを使用して再度アップロードしないようにし、今後の新しいアップロードには BYO CDN を使用できます。このセットアップが必要な場合は、お問い合わせください。また、複数の Video Cloud アカウントを作成し、それぞれに異なるコンテンツ配信戦略を設定することもできます。

BYO CDN またはリモートアセットを使用するコンテンツ配信の設定

ここでは、BYO CDN またはリモートアセットをセットアップするための主な手順を示します。

  1. CDN でアカウントを設定します (現在サポートされているアカウントについては、次のセクションを参照してください)。
  2. （ブライトコーブまたは外部の CDN から）移行する必要がある既存の動画コンテンツがある場合は、当社のサービスチームにお問い合わせください。
  3. セットアップを完了するには、Video Cloud アカウントマネージャにお問い合わせください。

ビデオクラウドが BYO CDN をサポートしているのはどの CDN ですか？

Video Cloud は、BYOB CDN コンテンツ配信用に以下の主要な CDN をサポートしています。

  • アカマイ
  • CloudFront
  • 断たせに
  • JOCDN
  • ライムライト

他の BYO CDN がサポートされる可能性がありますが、機能のサポートは異なります。詳細については、セールスまたはアカウントマネージャーにお問い合わせください。

BYO CDN でサポートされる機能

BYO CDN では、次の機能がサポートされています。

  • HTTP/HTTPS
  • HTTPSのみ
  • 署名付きマニフェスト URL
  • 署名付きコンテンツ URL（上記の CDN の場合のみ）
  • CDN 地理的制限 (CDN でサポートされている場合)
  • CDN IP 制限（CDN でサポートされている場合）
  • カスタム TTL 設定 (CDN でサポートされている場合)

ビデオクラウドレポートと BYO CDN またはリモート資産

BYO CDN またはリモートアセットを使用している場合、ダウンロードされたギガバイトは CDN と統合されていないため、動画やプレーヤーの Video Cloud Studio アナリティクスに表示されません。（表示分は Brightcove Player によって報告されるため、表示分が表示されます）。CDN プロバイダーから直接これらのレポートを取得する必要があります。「動画以外」コンテンツ、つまり、Video Cloud プレーヤーや画像などに発生した帯域幅の使用に関して、Video Cloud レポートにダウンロードされたギガバイトが表示される場合があります。
