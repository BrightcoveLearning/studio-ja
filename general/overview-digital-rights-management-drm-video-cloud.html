はじめに

デジタル著作権管理 (DRM) は、許可されているクライアントを除き、動画の再生を防止します。DRMの実装は様々ですが、クライアントが再生を許可するための何らかのキーを持っている場合にのみ、ビデオを暗号化して復号化するのが通常のメカニズムです。

次の項目も参照してください。

  • DRM によるコンテンツの取り込み
  • DRM 用のブライトコーブプレーヤーの設定
  • ネイティブプレイヤー SDK を使用したコンテンツセキュリティ (DRM)
  • DRM 出力保護設定

プレーヤー設定またはビデオプロパティで設定できるその他のコンテンツセキュリティメカニズムについては、以下を参照してください。

  • プレーヤーコンテンツ制限の設定
  • ビデオ可用性の設定
  • HLS 暗号化によるビデオの保護
ノート:
  • DRMまたはHLS暗号化を使用するには、アカウントでこれらの機能を有効にする必要があります。詳細については、アカウントマネージャーにお問い合わせください。
  • 動画が DRM で有効になっている場合、MP4 ソース/レンディションは Studio、CMS API、および再生 API で自動的に非表示になります（つまり、ビデオは存在しますが、ソースとしてプレーヤーに送信されません）。
  • Brightcoveは現在、ビデオのみまたはオーディオのみのDRM再生をサポートしていません。

サポートされているDRMタイプ

ビデオクラウドは、DRM の幅広いサポートを提供します。

  • パッケージング、ライセンス提供、再生を包括的にサポート
  • 特定のデバイスに対する適切な DRM の適応的使用:
    • Video Cloud はプラグインへの依存を最小限に抑えるためにネイティブデバイス DRM を好む
    • Video Cloud は、標準を活用して、クロスプラットフォームソリューションを提供するために必要なレンディションの数を減らします

ブライトコーブは、HTML5 ビデオ用に最も人気のある3種類の DRM をサポートしています。

  • Apple FairPlay
  • グーグルWidevineモジュラー
  • Microsoft PlayReady

複数の DRM タイプにサインアップして、すべてのデバイスおよびプラットフォームでコンテンツを保護できます。

    ブラウザー

    ブラウザー用の DRM は、デスクトップまたはモバイルウェブ (Android 6 以降のデバイスでは Chrome、iOS 11.2 以降のデバイスでは Safari) 用です。ブライトコーブは、Widevine Modular、Apple Fairplay、Microsoft PlayReady と共に MPEG-DASH と共通の暗号化を使用して、デスクトップ上に DRM を提供している。どのタイプが使用されるかは、ブラウザによって異なります。これにより、最新のブラウザで使用できるネイティブ DRM を活用しながら、異なるブラウザ間で再生できる一連のアセットが提供されます。

    iOSとAndroidアプリケーション

    以下の情報は、 Brightcove ネイティブプレイヤー SDK を使用して構築されたアプリに適用されます。SDK は、通常の再生に加えて、DRM でのオフライン再生もサポートしています。

    iOS

    iOSデバイスの場合、現在のVideo Cloud DRMソリューションはHLS with FairPlay Streamingである。

    アップルのFairPlay ストリーミング (FPS) 技術は、iPhone、iPad、Apple TV を含む Apple デバイスに HTTP ライブストリーミング (HLS) ビデオを配信するための安全なソリューションを提供します。確立されたセキュアキープロトコルを使用し、モバイルデバイスでの消費電力を最小限に抑えます。

    アンドロイド

    Androidデバイスの場合、ソリューションは MPEG-DASH + CENC（Widevine Modular）です。

    Google Widevine は、マルチプラットフォーム、マルチフォーマット DRM および動画最適化ソリューションを提供しています。コンテンツ所有者、インターネットビデオプロバイダー、小売業者、動画事業者は、インターネットに接続されたエンターテインメントプラットフォームにブランドの高品質のエンターテイメントを提供するために使用できます。

    Android デバイスの DRM セキュリティレベル

    Widevine は、CENC 暗号化、ライセンスキー交換、アダプティブストリーミング品質などの業界標準を実装し、ビデオ保護を管理します。Widevine は、受信デバイスのセキュリティ機能に基づいて、複数のレベルのストリーミング品質をサポートします。

    セキュリティのレベルは、L1、L2、および L3 という名前です。

    • レベル 1 -最高レベルのセキュリティレベル
      • クリアキーはセキュリティプロセッサによってのみ表示されます
      • 保護されたビデオパス。クリアビデオは公開されません。
    • レベル 2 -安全性が低い
      • クリアキーはセキュリティプロセッサによってのみ表示されます
      • 保護されたビデオパスがありません。クリアビデオが露出している。
    • レベル 3 -最低のセキュリティレベル
      • メインCPUによって行われるキー処理
      • 保護されたビデオパスがありません。クリアビデオが露出している。

    Android デバイスは、ハードウェアおよびソフトウェア構成に基づいて、L1 または L3 のセキュリティレベルをサポートします。L1 仕様を満たすデバイスのみが、Widevine で保護されたサーバーから HD またはそれ以上の品質のコンテンツをストリーミングできます。デバイスが L3 に準拠しているだけの場合は、DRM で保護された HD コンテンツをストリーミングできません。

    詳細については、以下を参照してください。

    • Widevine の DRM ソリューションドキュメント
    • Android デジタル著作権管理 (DRM) ドキュメント
    • Android デバイスのセキュリティレベルを決定する

    DRMでオフライン再生

    オフライン再生では、ユーザーは接続中にコンテンツをダウンロードして、オフライン時に視聴することができます。コンテンツを保護するために、ネイティブ SDK では以下を使用します。

    • Android 用ネイティブ SDK は Google Widevine を使用しています
    • iOS 向けのネイティブ SDK は FairPlay ストリーミングを使用しています

    詳細については、ネイティブプレーヤー SDK を使用したオフライン再生の概要を参照してください。

    その他のデバイス

    MPEG-DASHは、Android TVやChromecastを含む多くのデバイスでサポートされています。デスクトップブラウザ用のDRMでパッケージ化されたビデオは、これらのデバイスでも再生できます。

    多くの家電機器は、古い形式のDRMをサポートしています-スムーズなストリーミングでPlayReady

    DRM へのサインアップ

    DRMには追加料金が必要です。DRM 保護のバンドルについては、アカウントマネージャーにお問い合わせください。これにより、コンテンツを再生する場所を問わず保護できます。

    Apple FairPlay

    FairPlay Streaming（フェアプレイストリーミング）は、アップルが開発した、HLS形式の動画コンテンツ向けのDRMシステムである。

    FairPlay のアカウントを有効にするには、以下を行う必要があります。

    1. アップルからFairPlayライセンスを取得する
    2. Video Cloud アカウントの設定については、サポートにお問い合わせください

    既知の問題

    • モバイルデバイスでのFairplay再生で、外部画面が使用されている場合、オーディオは再生されますが、ビデオは黒い画像に過ぎません。

      詳細については、ネイティブプレイヤー SDK を使用したコンテンツセキュリティ (DRM) を参照してください。

    グーグルWidevineモジュラー

    Widevine Modular は、MPEG DASH形式の動画コンテンツを保護するためにGoogleが開発したシステムです。

    デフォルトでは、Video Cloud は Brightcove の Widevine ライセンスサーバーを使用します。

    マイクロソフトプレイレディ

    PlayReady（プレイレディ）は、マイクロソフトが開発したDRMシステムである。PlayReady は 2 つの形式があります。

    • Smooth PlayReady -スムーズなストリーミングコンテンツに DRM 保護を提供
    • PlayReady -MPEG-DASH コンテンツの DRM 保護を提供

    Video Cloudは、BrightcoveのPlayReadyライセンスサーバーを使用しています。

    Microsoftとのサービス契約で定義されているPlayReadyの同時実行制限があることに注意してください。見るこのMicrosoftドキュメント並行性の管理についての情報。動画の同時実行制限を超えると、最大数を超える視聴者の再生に失敗します。

    Video Cloudアカウントを設定する

    DRM は、Brightcove サポートおよびアカウントマネージャによってアカウントに設定する必要があります。詳細については、アカウントマネージャーにお問い合わせください。
