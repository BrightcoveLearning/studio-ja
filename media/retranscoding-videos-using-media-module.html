ブライトコーブは、Video Cloud アカウントに関連付けられたデフォルトの取り込みプロファイルを更新する権利を留保します。また、独自のカスタム取り込みプロファイルを更新する必要がある場合もあります。取り込みプロファイルが変更された場合、アカウント内の既存のビデオを再トランスコードして、新しい取り込みプロファイル設定に対応するレンディションを含めることができます。

注:既存のビデオを再トランスコードする機能は、ProfessionalアカウントとEnterpriseアカウントでのみ使用できます。この機能に興味がある場合は、アカウントマネージャーにお問い合わせください。

ビデオは、MediaモジュールまたはDynamic IngestAPIを使用して再トランスコードできます。を使用して動的取り込みAPI再トランスコーディングが必要なビデオが多数ある場合は、より良い解決策になる可能性があります。動的取り込み API を使用してビデオを再トランスコードするには、動的取り込み APIの概要開発者ドキュメントを参照してください。

注:アカウントと共有されている動画は再トランスコードできません。
注:再トランスコードできるのは、UploadモジュールまたはDynamic IngestAPIを使用してアップロードされた動画のみです。動画を再トランスコードできない場合、再トランスコードは失敗します。
注: 動画を再エンコードできるのは、動画がアカウントに追加されたときに動画マスターがアーカイブされた場合のみです。古い動画には、使用された取り込みプロファイルによっては、マスターがない場合があります。

動画を再トランスコードする手順

ビデオの既存のレンディションを置き換える手順は、次のとおりです。

  1. Media モジュールを開きます。
  2. 再変換したい動画をクリックします。
  3. [ ビデオファイル]セクションで、[ 再トランスコード] をクリックします。
  4. 使用する取り込みプロファイルを選択します。
  5. [ 既存のサムネイルとポスター画像を保持] をオンにして、既存の画像を保持します。このチェックボックスをオフにすると、選択した取り込みプロファイルの画像設定に基づいて新しい画像が生成されます。
    このチェックボックスをオンにすると、既存の画像が保持されることに注意してください-ただし、画像のURLが変更されます。
  6. [ 再トランスコード] をクリックします。

新しいレンディションが作成されている間、ビデオは既存のレンディションを使用して再生可能なままになります。
