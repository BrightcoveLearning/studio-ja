メディア共有を使用すると、パブリッシャーは動画を一度アップロードしてから、他のアカウントで動画を共有できます。アカウントには、自分のアカウント、法人アカウント、または他のパブリッシャーのアカウントを使用できます。共有した動画は、他の Video Cloud アカウントで個別に使用できます。通常、メディア共有が使用されるのは、パブリッシャーが動画コンテンツのマスターアカウントを維持しながら、組織の他の部門または子会社と動画を共有したい場合です。

ノート:
  • メディア共有は、Video CloudEnterpriseのお客様のみが利用できます。Video Cloud アカウントのアップグレードをご希望の場合は、ブライトコーブにお問い合わせください。
  • メディア共有は双方向にすることができます。各アカウントは、他のアカウントのマスターとアフィリエイトになることができます。

メディア共有は、CMS API を使用してプログラムで実装することもできます。詳細については、「CMS API を使用したメディア共有」を参照してください。

メディア共有の用語

このトピックでは、次の用語を使用します。

  • マスター -元のビデオを作成したアカウント。マスターはコンテンツを所有し、コンテンツを設定、管理、アフィリエイトに提供する責任があります。
  • アフィリエイト -動画を受信しているアカウント。アフィリエイトは、マスターから共有されたコンテンツを受け入れることができます。

メディア共有の制限

  • DRM を使用して保護されている動画の共有はサポートされていません。DRM 対応でないアカウントから DRM 対応のアカウントへのビデオの共有はサポートされていますが、共有ビデオは DRM 用にパッケージ化されません。

  • [ライブストリームを VOD に変換] オプションを使用して作成した動画では、共有に失敗します。これは、ライブストリームをクリップすることによって作成されたビデオのために動作します。

  • 共有ビデオは、元のビデオが可能な場合にのみ、オフラインで視聴できるようにダウンロードできます。このプロパティは元のビデオから継承され、共有ビデオに対してのみ変更することはできません。

  • SSAI で共有ビデオを再生する場合、SSAI マクロ置換では、子ビデオの代わりに親ビデオのメタデータが使用されます。SSAI は、親動画がとしてマークされている場合でもAdvertising='Free'、Ad Supported広告ルックアップをスキップします。

メディア共有関係の管理

アカウント間でビデオを共有するには、管理者／メディア共有設定の一部としてメディア共有設定を構成する必要があります。管理者メニューにアクセスできるのは、アカウント管理者のみです。他のアカウントと動画を共有する場合は、アフィリエイトを追加する必要があり、アフィリエイトアカウントはその関係に同意する必要があります。マスターアカウントとアフィリエイトアカウントは、いつでも関係を終了することができます。メディア共有関係を終了しても、アフィリエイトアカウントから共有動画は削除されません。アフィリエイトのアカウントから共有動画を削除する場合は、自分の（マスター）アカウントから共有動画を削除するか、ビデオのプロパティを開き、メディア共有セクションを開き、動画を削除するアフィリエイトアカウントのアイコン。

メディア共有設定を構成する方法については、「メディア共有設定の管理」を参照してください。

動画を共有する

デフォルトでは、Mediaモジュールにアクセスできるアカウント内のすべてのユーザーがビデオを共有できます。動画を共有するには、次の手順に従います。

  1. メディアモジュールを開き、1 つまたは複数のビデオを選択し、[その他] > [共有] の順にクリックします。
    注:多数の動画を共有する場合、ブライトコーブは一度に最大 50 本の動画を共有することをお勧めします。一度に多数の動画を共有すると、Studioでタイムアウトエラーが発生する場合があります。多数の動画を共有する必要がある場合は、 CMS APIを使用する必要があります。
  2. 動画を共有するアフィリエイトアカウントを選択し、[ 共有] をクリックします。検索フィールドを使用して、アカウント ID を検索できます。検索フィールドはあいまい検索をサポートしているため、ID の一部を入力すると一致が返されます。

動画を再共有（以前に共有した動画を同じアカウントで共有する）ことは可能です。これにより、アフィリエイトアカウントによって行われた動画メタデータの変更は、マスター (共有) アカウントの値に置き換えられます。動画を再共有するときは注意してください。

共有動画の確認

アフィリエイトアカウントには、マスターアカウントごとに受け入れモードを設定できます。承諾モードが Auto の場合、共有動画はアフィリエイトアカウントのメディアモジュールの動画リストに表示されます。いつ受け入れモードはマニュアル、 NS 保留中の株式共有ビデオの数を表示するリンクがメディアモジュールに表示されます。

注:受け入れモードの構成については、メディア共有設定の管理を参照してください。

[共有保留]リンクをクリックすると、共有動画のリストが表示されます。

共有保留リストから、以下ができます。

  • 拒否 -共有動画を拒否し、アカウントに含めない。共有動画を拒否しても、今後マスターがアカウントに再度共有できなくなるわけではありません。
  • 同意する -あなたのアカウントに共有ビデオを受け入れます。

クイックビューを使用すると、パネルを開いてビデオのプロパティを表示できます。共有ビデオを承諾すると、そのビデオは Media モジュールの [保留中の共有] リストに表示されなくなり、[すべてのビデオ] ビューに表示されます。

Media モジュールでの共有ビデオ

共有動画は、動画のサムネイルの上隅に共有アイコンが表示され、その動画がアカウント内またはアカウントから共有されたことを示します。

このアカウントが共有先である動画

共有アイコンにカーソルを合わせる（）共有の詳細が表示されます。

このアカウントが共有元である動画

共有アイコンにカーソルを合わせる（）共有の詳細が表示されます。

マスターアカウントは、動画のプロパティページの [ メディア共有 ] セクションを使用して、アフィリエイトアカウントと共有されている動画のステータスを確認できます。

エラーはここにも表示されることがあります。たとえば、[ カスタムフィールドの一致を強制する] をオンにすると、アフィリエイトアカウントにマスターアカウントと同じカスタムフィールドが必要です。そうしないと、共有は失敗します。

共有動画を削除する

共有動画がマスターアカウントから削除されると、共有した動画を削除すると、その動画を共有しているすべてのアフィリエイトアカウントから削除されることを知らせるメッセージが表示されます。

共有ビデオのプロパティ

動画がアフィリエイトアカウントで共有されると、次の動画プロパティが共有動画とともにコピーされます。

  • 名前
  • 短い説明と長い説明
  • 参照 ID（共有しているアフィリエイトアカウントに、同じ参照 ID の動画がすでに存在する場合、共有操作は失敗し、エラーメッセージが表示されます。これは、アフィリエイトが同じ参照 ID を使用できないようにする参照 ID 命名規則を設定することで回避できます)。
  • 関連リンク（リンクテキストとリンクURL）
  • タグ
  • ビデオ投影
  • サムネイルとポスター画像-デフォルトでは、マスタービデオのポスターとサムネイル画像はアフィリエイトアカウントで使用されます。マスターアカウントのポスターおよび/またはサムネイルに変更が加えられた場合、アフィリエイトアカウントが新しいポスターまたはサムネイル画像を作成していない限り、それらの変更はアフィリエイトアカウントに反映されます。アフィリエイトアカウントがポスターまたはサムネイル画像を変更すると、マスターアカウントで行われたポスター/サムネイルの変更は、アフィリエイトアカウントに反映されません。
  • テキストトラック-アフィリエイトは、共有動画のテキストトラックのプロパティを削除または編集することはできません
  • オーディオトラック-アフィリエイトは、共有動画のオーディオトラックプロパティを削除または編集することはできません
  • 広告 (無料/広告サポート)
  • 広告キー
  • スケジュール開始日と終了日
  • 地理的制限-パブリッシャーは、地理的制限に関して、マスターアカウントとアフィリエイトアカウントが同じように設定されていることを確認する必要があります。
  • キューポイントと関連するメタデータ
  • カスタムフィールド- マスターアカウントがカスタムフィールドの一致を強制するように設定されている場合、アフィリエイトアカウントにマスターアカウントと同じカスタムフィールドが必要です。そうしないと、共有は失敗します。
注:カスタムフィールドがリストタイプの場合、カスタムフィールドは両方の取引先で同じオプションを持つ必要があります。

共有ビデオには、マスタービデオが共有された時点で保持していたビデオメタデータフィールドのすべての値が含まれます。ただし、その時点から、マスターコピーとアフィリエイトコピーは独立したエンティティになります。

  • メディア共有を使用すると、ビデオファイルはコピーされず、ビデオメタデータのみがコピーされます。共有動画は、アカウント内の元の動画ファイルを指しています。
  • マスターアカウントから動画を削除すると、その動画はすべてのアフィリエイトアカウントから削除されます。
  • アフィリエイトが動画に加えた変更は、オリジナルまたは他の共有コピーには影響しません。アフィリエイトが共有動画を削除した場合、その動画はアフィリエイトのアカウントから削除されますが、マスターアカウントと他のすべてのアフィリエイトアカウントには残ります。
  • アフィリエイトは、Media モジュールを使用してテキストトラックを削除または更新することはできません。
  • アフィリエイトが動画を共有して受信すると、その動画はアクティブな状態になります。これは、ビデオがマスターアカウントでアクティブでない場合でも当てはまります。
  • アフィリエイトが動画を受け入れ、必須フィールドがない場合、動画は無効になります。
  • 共有動画を受信したアフィリエイトアカウントは、その動画を別のアカウントと共有することはできません。マスターアカウントのみがビデオを共有できます。
  • 各アカウントは、そのアカウントからの共有動画の再生（ビデオの再生と帯域幅の使用量）に対して個別に請求されます。
  • 共有動画に関するレポートは各アカウントに固有であり、各アカウントの動画の各コピーに関する情報を集計するものではありません。

動画を再共有（以前に共有した動画を同じアカウントで共有する）ことは可能です。これにより、アフィリエイトアカウントによって行われた動画メタデータの変更は、マスター (共有) アカウントの値に置き換えられます。動画を再共有するときは注意してください。

共有ビデオレンディション

トランスコーディングが完了する前に、動画をアフィリエイトアカウントと共有できます。共有動画を受信するアフィリエイトアカウントは、マスターアカウントで作成されたレンディションにアクセスできます。ソースアカウントの動画が再変換された場合、またはソースファイルが変更された場合は、共有動画を受信するアフィリエイトアカウントでこれらの変更が行われます。

アフィリエイトアカウントでは、共有動画のソースファイルを再変換または置き換えることはできません。

共有メディアと課金

アカウント間で共有されるメディアが課金にどのような影響を与えるかについて質問が頻繁に寄せられます。次に、パブリッシャー間で共有されるコンテンツに対する請求の処理方法について説明するシナリオをいくつか示します。

シナリオ 1

  • ブライトコーブ CDN のコンテンツを含むパブ A とパブ B
  • パブAはパブBにシェアする
  • パブ B はブライトコーブにストリームと帯域幅を支払う

シナリオ 2

  • BYOCDNを使ったパブ A
  • ブライトコーブ CDN または BYO CDN のパブ B (関係ありません)
  • パブAはパブBにシェアする
  • パブ B はブライトコーブにストリームを支払い、パブ A は帯域幅に対して独自の CDN を支払う

シナリオ 3

  • ブライトコーブ CDN のコンテンツを含むパブ A
  • BYOCDNを使ったパブ B
  • パブAはパブBにシェアする
  • パブ B は、課金パラメータが渡されていることを前提として、ストリームと帯域幅をブライトコーブに支払います。課金パラメータを適切に渡さないカスタムビデオプレーヤーの作成により、課金パラメータが渡されないエッジケースを見てきました。

シナリオ4-コンテンツはオリジンから取得する必要があります

 すべてのリクエストでアカウント ID を渡し 、各アカウントに独自のオリジンアクセスの請求が行われます。そのため、共有アカウントが頻繁にCDNキャッシュを逃し、コンテンツのオリジンに戻らなければならない場合は、その共有者に請求されます。Shareeアカウントが頻繁にCDNキャッシュを逃し、コンテンツのオリジンに戻らなければ、 Shareeアカウントに請求されます。オリジンアクセスの料金は、CDN 設定シナリオに関係なく別々になります。

シナリオ 5-イメージ

動的配信では、デフォルトでは、マスターアカウント（画像の作成者）は、共有アセットに関連付けられたすべてのイメージトラフィックに対して請求されます。アフィリエイトアカウントが共有コンテンツの新しい画像をアップロードまたはキャプチャした場合、画像トラフィックはアフィリエイトアカウントに関連付けられ、請求されます。
