ビデオのソースファイルを置き換えたい場合があります。一般的なケースの 1 つは、すでに公開されているビデオを編集する必要がある場合です。動画を削除して再アップロードすると、すべての動画メタデータを再入力する必要があります。また、動画に関連付けられている動画分析はすべて失われます。

新しいレンディションが作成されている間、既存のレンディションを使用して動画を再生できます。

注:アカウントと共有されている動画のソースファイルを置き換えることができません。
注:ビデオファイル名（拡張子を含む）は、120 シングルバイト（60 全バイト）文字を超えないようにしてください。そうした場合、ビデオは正常に取り込まれますが、後で再トランスコードすることはできません。

動画ソースファイルを置換する手順

ビデオソースファイルを置き換えるには、次の手順に従います。

  1. ナビゲーションヘッダーの [ メディア ] をクリックして、Media モジュールを開きます。
  2. ソースファイルを置き換えるビデオのリンクをクリックします。
  3. [ ビデオファイル]セクションを見つけて、[ ソースファイルを置換] をクリックします。
  4. アップロードモジュールが別のタブで開きます。アップロードモジュールが、ソースファイルを置き換えることを示していることを確認します。
  5. 使用する取り込みプロファイルを選択します。
    注:動的配信またはコンテキスト対応インジェストプロファイルを使用して最初にトランスコードされた動画は、これらの取り込みプロファイルタイプのいずれかを使用してのみ再変換できます。レガシー取り込みプロファイルを使用してトランスコードされた動画は、あらゆるタイプのプロファイルで再トランスコードできます。
  6. Uploadモジュールを使用して、新しいソースファイルをアップロードします。
  7. [ 既存のサムネイルとポスター画像を保持] をオンにして、既存の画像を保持します。このチェックボックスをオフにすると、選択した取り込みプロファイルの画像設定に基づいて新しい画像が生成されます。
    このチェックボックスをオンにすると、既存の画像が保持されることに注意してください-ただし、画像のURLが変更されます。
  8. 元のソースファイルに関連付けられているオーディオトラックはすべて、新しいものに置き換えられます。チェック既存のオーディオトラックを保持する取り込まれた個別のオーディオトラックを保持します。
  9. [処理]をクリックして、トランスコードを始めます。「処理完了」というメッセージが表示されます。更新されたレンディションが使用できるようになるまでに遅延が発生することに注意してください。メディアモジュールの [ ビデオファイル ] セクションの [ アップロード日]の値をチェックして、新しいレンディションが使用可能であることを確認します。
