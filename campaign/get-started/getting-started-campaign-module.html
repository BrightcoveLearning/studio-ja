キャンペーンモジュールは、キャンペーン対応のBrightcoveプレーヤー内で表示されるビデオのビデオエンゲージメントデータを追跡するために使用されます。閲覧データは Video Cloud に保存され、必要に応じて、Eloqua、Marketo、HubSpot、Salesforce などの一般的なマーケティング自動化プラットフォームと同期することができます。

キャンペーンモジュールにアクセスするには、Video Cloudにログインして、運動ナビゲーションヘッダー内。キャンペーンのホームページには、動画の視聴時間、視聴された動画、視聴者に関する洞察を提供する動画エンゲージメントデータが表示されます。これによって、視聴者のエンゲージメントとコンバージョンを改善できます。

ホームページでは以下ができます。

  • 動画のパフォーマンスを見る
  • リードのパフォーマンスの表示
  • キャンペーンモジュールのステータスを表示する

キャンペーンモジュールは、次の機能を提供します。

  • データ接続のセットアップ
  • キャンペーンプレーヤーを作成する
  • リードフォームの作成と管理
  • 電子メールにパブリッシュ
  • プレイヤーイベントを表示する
  • ビューアプロファイルの表示
  • 同期アクティビティの表示
  • キャンペーン設定を構成する
  • キャンペーンデータのエクスポート
  • ギャラリーのリードをエクスポート
  • ビューアデータを管理する

動画パフォーマンスを表示する

ビデオパフォーマンスセクションは、キャンペーン対応プレーヤー内で再生されたビデオの高レベルのビデオパフォーマンスを提供します。

注:キャンペーン分析システムは、一般的なビデオクラウド分析システムとは別のものであり、2つの追跡データは異なります。したがって、2 つを比較すると、いくつかの指標に違いが見られます。たとえば、Video Cloud分析システムは、ビデオが開始されるとすぐに視聴回数をカウントしますが、Campaign分析システムは、ビデオが開始されてから10秒まで視聴回数をカウントしません。したがって、14 人の視聴者が動画を開始したが、そのうちの 6 人が動画を 10 秒再生する前に停止した場合、Video Cloud アナリティクスでは 14 のビューが表示され、Campaign モジュールには 8 が表示されます。

を選択することから始めます期間チャート用。次のオプションを使用できます。

  • 7日間
  • 30日間
  • 90日

次のグラフが表示されます。

  • 最も視聴された動画
  • 最も魅力的な動画
  • ビューアの傾向

[視聴回数の多い動画]:

最も視聴された動画のグラフには、選択した期間にキャンペーンが有効なプレーヤーで最も視聴された動画が表示されます。

最も魅力的な動画

最も魅力的な動画のグラフには、選択した期間にキャンペーンが有効なプレーヤーで最も魅力的な動画が表示されます。エンゲージメントとは、視聴された割合の平均です。動画エンゲージメントの計算方法の詳細については、トレーニングビデオの「動画エンゲージメントスコアの計算」をご覧ください。

視聴者の傾向

[視聴者の傾向] グラフには、選択した期間におけるビデオの視聴回数が表示されます。既知のビューアと匿名ビューアの両方からのビューが表示されます。グラフをポイントすると、実際のデータポイントが表示されます。

注:Marketa Munchkin 接続を使用する場合、視聴者の傾向グラフには不明な視聴者のデータのみが表示されます。Munchkin 接続を使用する場合、ユーザーの詳細は収集されません。より詳細なユーザー情報については、Marketa REST 接続を使用する必要があります。詳細については、REST API を使用したビデオクラウドと Marketo の統合を参照してください。

リードのパフォーマンスの表示

リード情報を表示するには、[ リード]タブをクリックします。[リード]タブは、アカウントにキャンペーンリードフォームが作成されている場合にのみ表示されることに注意してください。リードパフォーマンスセクションは、キャンペーン対応プレーヤー内で再生されたビデオの高レベルのリードパフォーマンスを提供します。を選択することから始めます期間チャート用。次のオプションを使用できます。

  • 7日間
  • 30日間
  • 90日

以下に基づいたグラフが表示されます。

  • キャンペーンファネル
  • 動画の変換トップ
  • リードチャート

キャンペーンファネル

キャンペーンファネルは、選択した期間のキャンペーンリードフォームを使用して、キャンペーンが有効なプレーヤーの動画の指標を表示します。

  • 動画のインプレッション -読み込まれた動画の数
  • 動画再生回数 -ストリーミングの開始時に録画された動画の再生回数（巻き戻しまたは再生は含まれません）。個々の視聴者の測定値ではありません。
  • 引合フォームビュー -引合フォームビューの合計数
  • コンバージョン -送信されたリードフォームの合計数

動画の変換トップ

コンバージョン率の高い動画チャートには、コンバージョン率が最も高い動画が表示されます。コンバージョン率は、送信されたリードフォームの数を動画のインプレッション数で割って算出されます。

リードチャート

「引合チャート」には、選択した期間における発行済引合フォームの数が日別に表示されます。グラフをポイントすると、実際のデータポイントが表示されます。

キャンペーンモジュールのステータスの表示

[キャンペーンステータス]セクションには、キャンペーンモジュールの現在のステータスが表示されます。

データ接続の設定

NS データ接続オプションは、キャンペーン統合を構成する場所です。キャンペーンは、一般的なマーケティング自動化プラットフォーム（ "MAP  統合」）または表示データをローカルに保存してから、Audience API（「非MAP」統合）を使用してエクスポートまたは抽出できます。次の表に、サポートされている統合の概要を示します。

MAP 統合 マップ以外の統合
  • オラクル・エロクア
  • MarketO (REST API)
  • マルケト (Munchkin)
  • セールスフォース
  • HubSpot (REST API)
  • HubSpot (クライアント側 API)
  • カスタム統合
  • シングルサインオン
  • キャンペーン追跡

キャンペーン対応のプレーヤーを公開するときは、高度な埋め込み（ページ内）公開コードを使用してください。キャンペーンプラグインはiframe内で実行されるため、標準埋め込み（iframe）公開コードは機能しません。そのため、親ページのURLパラメーターにアクセスできず、正しいCookieを取得できない可能性があります。

注:設定できる接続の数は、Video Cloud サブスクリプションに基づいています。

キャンペーンプレーヤーの作成

NS キャンペーンプレイヤーオプションを使用すると、キャンペーン対応のBrightcoveプレーヤーを作成できます。ビデオエンゲージメントデータは、キャンペーン対応のビデオプレーヤー内でビデオコンテンツが表示された場合にのみ記録されます。キャンペーン対応プレーヤーの作成については、を参照してください。キャンペーン対応プレーヤーの作成。

リードフォームを作成、管理する

[ リードフォーム ] オプションを使用すると、データ取得用のリードフォームを作成できます。リードフォームが作成されると、キャンペーンが有効なプレーヤーに関連付けることができます。リードフォームの作成については、を参照してください。キャンペーンの作成 リードフォーム。サードパーティのマーケティング自動化プラットフォーム内で作成されたリードフォームを作成し、使用することもできます。

動画をメールに公開する

[ 電子メールに公開]オプションを使用すると、プレーヤーとビデオを選択し、HTML 埋め込みコードを生成してビデオを電子メールで送信できます。詳細については、を参照してください。キャンペーンモジュールを使用してビデオを電子メールに公開する。

注:Media モジュールには、ビデオを電子メールで公開する機能もあります。詳細については、「ビデオを電子メールに公開する」を参照してください。

プレーヤーイベントを表示する

[ すべての最近のアクティビティ ] オプションでは、過去 24 時間に記録されたプレーヤービューイベントを表示できます。詳細については、を参照してください。キャンペーンモジュールでのプレーヤーイベントの表示。

ビデオアクティビティの表示

[ ビデオ (マップ以外) ] オプションを使用すると、ビデオの表示アクティビティを表示できます。詳細については、「ビデオアクティビティの表示」を参照してください。

ビューアプロファイルの表示

Viewer Profile (マップ以外)オプションを使用すると、プレイヤービューイベントをビューアごとに表示できます。このオプションは、SSOまたはカスタム統合、あるいはキャンペーントラッキングを使用している場合に有効になります。詳細は、「ビューアプロファイル情報の表示」を参照してください。

同期アクティビティの表示

NS 同期アクティビティオプションは、キャンペーンからマーケティング自動化プラットフォームに同期された最後の10個のジョブを表示します。同期は時間単位で実行されます。接続を選択し、[ 同期のテスト ] ボタンをクリックして同期を実行します。これは、新しい統合をテストし、イベントデータがマーケティングオートメーションプラットフォームに送信されるかどうかを待たずに済むようにする場合に役立ちます。テスト同期は、5 分に 1 回、1 時間に 10 回まで実行できます。

注:カスタム統合を使用する場合、同期ジョブは作成されません。同期ジョブは、マーケティングオートメーションプラットフォームとの統合を使用する場合にのみ作成されます。

キャンペーン設定の構成

[ 設定]オプションでは、アカウントのデフォルト設定を構成できます。詳細については、を参照してください。キャンペーン設定の構成。

キャンペーンデータのエクスポート

NS キャンペーンデータのエクスポートオプションは、ビューイベントとリードフォームデータをエクスポートするために使用されます。このオプションは、アカウント管理者に対してのみ表示されます。キャンペーンデータのエクスポートについては、を参照してください。キャンペーンモジュールからのキャンペーンデータのエクスポート。

ギャラリーリードのエクスポート

[ ギャラリーリードのエクスポート ] オプションは、ギャラリーエクスペリエンスから取得したリードフォームデータをエクスポートするために使用します。このオプションは、アカウント管理者に対してのみ表示されます。ギャラリーリードデータのエクスポートの詳細は、「ギャラリーリードデータのエクスポート」を参照してください。

ビューアデータの管理

[ ビューアデータの管理 ] オプションでは、ユーザーを検索し、そのユーザーを削除するか、そのユーザーのデータをエクスポートできます。このオプションは、アカウント管理者に対してのみ表示されます。ビューアデータの管理の詳細は、「ビューアデータの管理」を参照してください。
