ある

アクティブメディア
アカウント内のアクティブな動画の数です。このデータはリアルタイムではありません。ビデオライブラリから 1 日 1 回同期されます。また、「リアルタイム」分析データ (3 日未満) を見ている場合、このデータは正確ではない場合があります。
アナリティクス API
REST ベースのアプリケーションプログラミングインターフェイスで、開発者は Video Cloud 動画の分析指標を JSON 形式で直接取得できます。NS Analytics API Video Cloud StudioのAnalyticsUIのデータを提供します。
アクセストークン
CMS API、分析 API、動的取り込み API などの REST API へのアクセスを許可する一時的な OAuth2 トークン。
アフィリエイト
Video Cloud プレーヤーをシンジケートするウェブパブリッシャーです。
広告リクエスト
Video Cloud プレーヤーから広告サーバーへのリクエスト（呼び出し）。広告リクエストは HTTP 呼び出しで、広告サーバーがコンテキストに適した広告をターゲットにするのに役立つ情報を渡すキー/値のペアが含まれます。すべての広告リクエストで広告が返されるわけではありません。
アナリティクス API
NS Analytics API VideoCloudアカウントとビデオの分析データをプログラムで取得できます。
アップルHLS
Apple HTTP ライブストリーミング。オンデマンドでビデオを配信するために Apple によって実装されたプロトコル。Apple HLS は、ビデオを小さなファイルダウンロードのシーケンスに分割し、各ダウンロードは HTTP を使用してビデオの短いチャンクをロードします。Apple HTTP ライブストリーミングはライブストリーミングビデオとは関係ありません。
アスペクト比
ビデオのフレームの幅と高さの比率。標準システムでは 4:3、HDTVの場合は 16:9。

B

帯域幅
オーディオ機器またはビデオ機器がエンコードまたはデコードできる信号周波数の範囲。ビデオはオーディオよりも高い周波数を使用するため、広い帯域幅が必要です。
ブログ
定期的な投稿で構成された Web ベースの出版物。多くの場合、時系列の逆順になっています。
ブライトコーブプレーヤー
Brightcove Player は、高パフォーマンスのクロスプラットフォーム HTML5 ファースト動画プレーヤーで、すばやくロードでき、デスクトップおよびモバイルプラットフォームにわたって高品質のアダプティブビットレート HLS 動画を配信し、お気に入りの広告や分析システムと統合できます。
ブライトコーブプレーヤー API
Brightcove Player API を使用して、ブライトコーブプレーヤーの外観と動作をプログラム的に制御できます。
ブロードバンドビデオチャンネル
インターネットを通じて配信される関連動画の集まり。
バンパー
フルレングスのビデオの前またはプレーヤーのロード時に表示される短いビデオクリップ。

C

CMS API
NS CMS APIビデオメタデータへのキャッシュされていない読み取りおよび書き込みアクセスを提供します。
Codec
アナログビデオまたはオーディオをデジタル形式でキャプチャするために使用されるアルゴリズム。と比べて容器。
コンテナ
ビデオコンテナは、ビデオのメタデータをビデオおよびオーディオデータとともに格納する方法を定義する形式です。これは、データのコーディングや圧縮を定義しません。たとえば、MP4 は H.264 ファイルのコンテナにすることができます。
コンテンツ配信ネットワーク (CDN)
エンドユーザーにコンテンツ（特に大きなメディアコンテンツ）を透過的に配信するネットワークサーバを提供するサービス。著名なCDNはアカマイとライムライトの2つです。
コンテンツターゲティング
コンテンツターゲットは、その訪問者に関する既知の情報に基づいて、異なる訪問者に異なるコンテンツを表示するために使用されるプロセスです。
CSS
カスケードスタイルシートは、HTML ページのデザインを制御するための標準的なメカニズムです。CSS は、ブライトコーブプレーヤーのスタイルを設定するために使用されます。
キューポイント
動画の特定のポイントに挿入される非表示マーカー。ミッドロール広告リクエストやクローズドキャプションなどの外部イベントをトリガーするために使用できます。Media モジュールを使用してキューポイントを設定できます。

D

ダッシュ (MPEG-DASH)
HTTP 経由の動的アダプティブストリーミング (MPEG-DASH とも呼ばれます) は、従来の HTTP Web サーバーから配信されるインターネット経由のメディアコンテンツの高品質ストリーミングを可能にするアダプティブビットレートストリーミング技術です。AppleのHTTPライブストリーミング（HLS）ソリューションと同様に、MPEG-DASHはコンテンツを小さなHTTPベースのファイルセグメントに分割することで機能します。
ドメイン制限
指定されたドメイン以外でプレーヤーが機能することを妨げるコンテンツセキュリティの基本的な形式。
DRM
デジタル著作権管理は、メディアファイルを暗号化してパッケージ化し、ポリシーキーに基づいてのみ復号化できるようにする方法です。
動的配信
マルチビットレートストリーミングを参照してください。
動的取り込み API
ダイナミックインジェスト（DI）API は、ビデオアセットを取得してレンディションを作成するように Video Cloud に指示するために使用されます。NS 動的取り込みAPIキャプション、チャプター、またはサブタイトルのサムネイルおよびポスター画像またはWebVTTファイルを取り込むためにも使用できます。

E

エンコーディング
ビデオおよびオーディオデータをプレーヤーで再生できるストリームに変換します。
暗号化
許可された当事者のみがそれを見ることができるようにビデオをエンコードするプロセス。

F

フレームレート
ビデオクリップが表示されるフレーム数/秒です。

G

ジオフィルタリング
多くの場合、権利所有権の理由により、動画の地理的な制限を課します。

H

H.264フォーマット
MPEG-4 AVC (Advanced Video Coding) とも呼ばれる。インターネット経由でビデオを配信するために使用されるファイル形式。オンライン動画配信にはH.264形式がおすすめです。H.264 ファイルは通常 MP4 コンテナを使用し、ファイル拡張子は.mp4 です。以前の形式よりも大幅に圧縮率が高くなります。
HLS
HTTP ライブストリーミング-クライアント側でストリームされる短いセグメントでビデオを利用できるようにする Apple によって作成されたビデオフォーマット。
HTML
(HyperText Markup Language) Web ブラウザに表示されるハイパーテキストやその他の情報を含む Web ページを作成するために設計されたマークアップ言語。
HTML5
HTML5 は、World Wide Web のコンテンツを構造化および提示するために使用されるインターネットの中核技術マークアップ言語です。HTML5は、World Wide Web Consortium (W3C) のHTML規格の第5改訂版である。

おい

プロファイルの取り込み
取り込みプロファイルには、アップロードモジュールまたは動的取り込み API を使用して動画をアップロードするときに使用されるトランスコード設定が含まれています。
プロファイルの取り込み API
取り込みプロファイル API を使用すると、Video Cloud アカウントの取り込みプロファイルの作成、取得、更新、削除を行うことができます。取り込みプロファイルは、ビデオを Video Cloud に追加するときに作成されるレンディションを定義します。

J

JavaScript
他のアプリケーションに埋め込まれたオブジェクトへのスクリプトアクセスを可能にするためにも使用されますが、ウェブサイトでの使用で最もよく知られているスクリプト言語です。
JPEG (共同写真専門家グループ)
デジタル画像に使用される非可逆圧縮の一般的に使用される標準的な方法です。JPEG は、サムネイル、動画静止画、背景画像用に Brightcove サービスで使用される標準的な画像形式です。

K

キー/値のペア
プレーヤーが広告リクエストを送信すると、広告サーバーに広告のコンテキストに関する情報を提供し、広告のターゲティングを改善する 1 つ以上のキーと値のペア（カテゴリ=スポーツなど）が含まれます。

L

レターボックス
画像の上下にスペース（通常は黒いバー）を残して、異なる縦横比の画面上にワイドスクリーンビデオを表示する方法。
ロゴ
会社のブランドを象徴的に表す独自のグラフィック。
ロゴオーバーレイ
ビデオ再生ウィンドウに表示される小さな透明なイメージ。デジタルオンスクリーングラフィック (DOG)、バグ、透かしとも呼ばれます。
ループ/ループ
プレイリストの動画が完成し、再生リストの再生が開始されたとき。
ライブストリーミングビデオ
VOD のようにストリーミング用にエンコードされたファイルを配布するのではなく、キャプチャされたストリームとエンコードされたストリームを直接プッシュすることにより、イベントのビデオをストリーミングするためのプロトコル。

M

メディアライブラリ
Video Cloud アカウントの動画、再生リスト、およびその他のアセットのコレクション。
メタデータ
メタデータとは、動画の定義においてユーザーが提供する説明、タグ、ジャンル、およびその他の関連情報を指します。
MPAAフィルム・レーティング・システム
その内容に基づいて映画を評価するために、アメリカ映画協会によって制定されたシステム。MPAAレーティングシステム（G、PG、Rなど）は、攻撃的な可能性のあるコンテンツを分類するための最も認知されているシステムですが、MPAAが個々のレーティングに商標を持っているため、通常は映画業界での使用が制限されています。
マルチビットレートストリーミング
複数のバージョンをサポートする機能、またはレンディション単一のビデオで、視聴者のブラウザのサイズと帯域幅に最も一致するレンディションを配信できます。以前は「動的配信」と呼ばれていました。「
マルチシングとデマルチシング
さまざまなものとの間のオーディオおよびビデオストリームの書き込みと読み取り 容器フォーマット。

N

ネイティブプレイヤー SDK
ブライトコーブのネイティブプレーヤー SDK には、iOS および Android 用の動画対応アプリを構築するために必要なツールが用意されています。

お

OAuth
OAuth は承認のためのオープンスタンダードです。OAuth は、クライアントアプリケーションに、' リソースオーナーの代わりにサーバーリソースへの「安全な委任アクセス」を提供します。
OAuth API
OAuth API は、認証トークンを取得してブライトコーブの他の REST API にアクセスできる RESTful API です。

うP

再生する
ビデオクラウドプレーヤーでのビデオメディアの再生です。
再生 API
Playback API を実行時に使用して、モバイルアプリまたはビデオポータルのビデオおよびプレイリストデータにアクセスできます。
プレーヤー管理 API

プレーヤー管理 API は、次の目的で使用できます。

  • プレーヤーを作成する
  • プレイヤーの設定を表示する
  • プレイヤーの設定を変更する
  • 出版社に関連付けられているすべてのプレーヤーを一覧表示します
  • プレーヤーを公開してコンシューマーが使用できるようにする
プレーヤー公開コード
Brightcove Player を Web ページに埋め込むために使用されるコード「スニペット」。
プレイリスト
Video Cloud プレーヤーで再生するために、特定の順序でグループ化されるビデオのコレクション。
プレイリスト API
プレイリスト API は、JavaScript を使用してプレイリストをプログラム的に制御するために使用されます。
ポリシー API
ポリシー API は、 Playback API へのアクセスに使用できるポリシーキーを作成または取得するために使用されます。ポリシーキーは、ブライトコーブプレーヤー用に自動的に生成されます。
ポスター画像
ビデオ再生の前または後にビデオプレーヤーの再生領域に表示される静止画像。
px
CSS で使用されるピクセルの略語。

Q

 
 

R

レンディション
マルチビットレートストリーミングを使用する場合、同じビデオの複数のバージョンを作成し、それぞれ異なるビットレートとサイズを設定できます。これらの各バージョンは、レンディションと呼ばれます。Video Cloud マルチビットレートストリーミングは、プレーヤーのサイズと視聴者の接続速度に基づいて、最適なレンディションを選択します。
参照ID
パブリッシャーが動画または再生リストに割り当てた一意の ID。参照 ID（参照 ID）は、ビデオ ID または再生リスト ID とは区別されます。これは、Video Cloud によって生成される一意の ID です。参照 ID を使用すると、Video Cloud と他のサービス（パブリッシャーのコンテンツ管理システムなど）との調整が可能になります。
RSS フィード
Really Simple Syndication は、Web サイト、ブログ、インターネットテレビチャンネルのコンテンツの更新をユーザーに通知するための一般的なテクノロジです。

S

検索エンジン
ウェブ上の情報を見つけるために設計されたGoogleなどのプログラム。
ソーシャルネットワークのウェブサイト
Facebook や Twitter などの Web サイト。視聴者のメンバーは、メッセージ、メディア、その他の情報を互いに共有できます。
ストリーミング
ネットワークおよびインターネット経由での使用を目的とした、低ビットレートのエンコード形式。ストリーミングでは、ユーザーの接続速度に合わせてエンコードされたビットレートで小さなセグメントにビデオが配信されるため、リモートビューアは、ビデオファイル全体をダウンロードしなくても、最小限の停止でオーディオまたはビデオを再生できます。

T

タグ
コンテンツを整理するための個々のキーワードまたはフレーズ。ブライトコーブでは、パブリッシャーが動画を整理して分類したり、プレイリストを作成したりするための手段として、へのタグの使用をサポートしています。
サムネイル
プレーヤー、Web サイト、および動画検索エンジンの動画のリストで使用される動画に関連付けられた小さな画像。
トークンベースの認証
トークンベースの認証システムでは、ユーザ名とパスワードを入力することで、ユーザ名とパスワードを必要とせずに特定のリソースを取得できるトークンを取得できます。トークンを取得すると、ユーザーはトークンをリソースに提供できます。トークンは、指定された期間の特定のリソースへのアクセスを提供します。
テレビのペアレンタルガイドライン
米国連邦通信委員会（FCC）がコンテンツの適切な年齢に基づいてテレビ番組を分類するために制定したコンテンツレーティングシステム。

う

URL
ユニフォームリソースロケータ (URL) は、インターネット上のリソースとその場所を識別する方法です (http://www.brightcove.com など)。

V

可変ビットレート
入力材料の各ユニットを異なるサイズに圧縮できる圧縮方式。たとえば、MPEG-2 ビデオの場合、これは「簡単な」シーケンス（モーションなし）が非常に小さいサイズに圧縮できるのに対し、「ハード」シーケンス（モーションやシーンカットが多い場合）ははるかに大きなサイズに圧縮できることを意味します。
ビデオ

Video Cloud プレーヤーで再生する 1 つのビデオタイトル。ビデオはビデオファイルに基づいており、サポートされている多くのビデオファイル形式の 1 つとして開始できますが、H.264 ファイルまたは FLV ファイルにまだそれらの形式でない場合は、トランスコードされます。さらに、ビデオには、次のようなメタデータを含めることができます。

  • 名前
  • 簡単な説明
  • 長い説明
  • タグ
  • 動画静止画
  • ポスター画像
  • バンパー
  • ロゴオーバーレイ
ビデオクラウドスタジオ

Video Cloud サービスにアクセスして制御するためのインターネットアプリケーション。Studioには、さまざまなタスクを完了するための一連のモジュールが含まれています。

  • アップロード -Video Cloud アカウントに動画をアップロードする
  • メディア -ビデオのプロパティの設定、フォルダーを使用したビデオの整理、ビデオの公開
  • プレーヤー -プレーヤーの作成とスタイル設定を行い、プレーヤーの公開コードを生成する
  • アナリティクス -動画とプレイヤーのパフォーマンスに関するアナリティクスを表示
  • ライブ -ライブイベントの作成とストリーミング
  • ギャラリー -ビデオコンテンツ用のビデオポータルの作成
  • オーディエンス -Video Cloud からマーケティングオートメーションプラットフォームに動画分析を接続
  • ソーシャル -YouTube、フェイスブック、Twitterに動画を配信する
ビデオ編集ツール
パーソナルコンピュータ上でビデオファイルを編集するためのソフトウェアアプリケーション。
ビデオポスター画像
ビデオ再生の前または後にビデオプレーヤーの再生領域に表示される静止画像。
VOD
ビデオオンデマンド。ストリーミングまたはプログレッシブダウンロードによって、配信されたビデオファイルをオンデマンドで再生できるプロトコル。とは対照的にライブストリーミングビデオ。

W

WebVTT
Web ビデオテキストトラック形式。WebVTT ファイルは、ビデオコンテンツのキャプションや字幕、テキストビデオの説明、コンテンツナビゲーション用のチャプター、オーディオコンテンツまたはビデオコンテンツと時間調整されたメタデータの形式を提供します。ブライトコーブプレーヤーは WebVTT 形式のキャプションを使用します。

X

XLSX
マイクロソフトエクセルファイル。分析データは、この形式でエクスポートできます。

Y

 
 

Z
