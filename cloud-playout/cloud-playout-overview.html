はじめに

Brightcove Cloud Playout は Studio モジュールで、VOD カタログから予定されたアイテムの再生をライブフィードと組み合わせて、ブライトコーブ Live の信頼性と拡張性を備えたリニアチャンネルとしてストリーミングすることができます。

クラウドプレイアウトは、スケジュールされたプログラミングのチャンネルのための強力な、エンドツーエンドソリューションを提供しています, 含みます:

  • 24x7 リニアチャンネル
  • ポップアップ/仮想チャネル
  • シミュレートされたライブイベント

クラウドプレイアウトは、事前に録画されたコンテンツを活用したブロードキャストのようなライブTV体験を通じて、オーディエンスのエンゲージメントを高め、新たな収益機会を創出するのに役立ちます。既存の VOD アセットを使用して、スケジュールされたライブチャンネルを作成します。

特徴

クラウドプレイアウトの機能は次のとおりです。

  • ビデオクラウドメディアライブラリから VOD アセットをインポートします。
  • VOD アセットとスケジュールされたライブフィードを切り替えます。
  • 迅速かつ信頼性の高い方法で、スケジュール変更を直前までリアルタイムで対応できます。
  • サーバーサイド広告挿入（SSAI）でチャンネルのコンテンツを収益化します。
  • プログラムをループするようにチャンネルを設定します。
  • デバイスごとに再生を最適化します。
  • ドラッグ＆ドロップで、プログラムスケジュールをいつでも並べ替えます (Running)。
  • ブライトコーブ Live、Amazon S3 バケット、RTMP 出力経由で所有および運営されているプロパティにチャネルをプッシュします。
  • チャンネルのEPG（電子プログラムガイド）を取得するためのAPI

制限事項

次の前提条件と制限が適用されます。

  • 動画は、動的配信プロファイルを使用して取り込みまたは再変換し、MP4レンディションが必要です。
  • クラウドプレイアウトでサポートされる最大入力ビットレートと解像度は 20Mbps/1080P です。これは、VOD コンテンツとライブ入力の両方に適用されます。
  • ブライトコーブでは、動画と電子プログラムガイドの正確性を正しく移行するために、2 分以上の動画の使用をお勧めします。
  • 現在、動画に追加されたキャプション（テキストトラック）は、クラウド再生チャンネルには表示されません。キャプションのサポートは、将来のリリースで追加される予定です。
  • ライブイベントの最小長は 5 分です。
  • クラウドプレイアウトのすべてのライブイベントは、Video Cloud リモートアセットと 1:1 のリレーションを共有します。このリモートアセットは、他のチャンネルでは再利用できません。ライブイベントがチャンネルから削除されると、関連付けられたリモートアセットは自動的に削除されます。チャンネルを削除すると、関連するリモートアセットも自動的に削除されます。
  • 別のアカウントからアカウントと共有されている動画は、現在 Cloud Playout チャンネルに追加できません。
  • ライブイベントが現在再生されているのにアクティブなフィードがない場合、スレートコンテンツは、ライブフィードが再開するかイベントが終了するまで、ループで再生されます。スレートMP4が再生を開始する前に、約30秒の遅延があります。
  • Video Cloud アカウントが地理制限用に設定されている場合、チャネル出力リージョンは Video Cloud地理制限ポリシーと互換性のあるリージョンに作成する必要があります。
  • 最高のエクスペリエンスを得るには、BrightcoveはGoogleChromeブラウザの使用をお勧めします。
  • SSAI対応チャネルの場合、宛先を広告対応RTPライブイベントに設定する必要があります。
